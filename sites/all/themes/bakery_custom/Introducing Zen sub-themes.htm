Lesson Goals:

  • Learn about Drupal's theming engine with sub-themes

A good way to learn about creating themes is by modifying existing ones. But, as we know, we don't hack core and we don't change contrib modules, and for the same reasons it's better not to directly change the files included in a downloaded theme. Instead we want to create our own theme which just uses and alters the functionality of another theme, similarly to the support modules we have been creating this week. This is called a sub-theme.

Sub-Theme from a Base Theme

A sub-theme inherits all properties from a base theme. The base theme is your starting point - perhaps we just want to change the colors a bit, add a new background and some better fonts. This will allow you to modify something that already exists rather than starting from scratch. But it also allows you the benefits of keeping your code separate from the base theme so that the base theme can be updated safely without causing conflicts with your code.

Zen

We're going to work with creating a sub-theme based around a theme called Zen. Why use Zen? Here is some discussion on that point. To summarize:

  • It gives you a clean, minimal starting point to base your theme on.
  • It also lets you have a reusable workflow, similar to what larger shops use because you are always starting from the same set of files.
  • Search Engine Optimization (SEO) written in already so that the content of the site is easy to find with a search engine.

Here are the steps to create a sub-theme using Zen as your base.

  • Download the Zen theme and put it in your sites/all/themes folder
  • Copy out the STARTERKIT folder from zen into sites/all/themes and rename the folder with the name of your sub-theme - for example, myzen. Just like modules, themes are named lowercase.
  • Change the word STARTERKIT to the name of your sub-theme, myzen, everywhere in the files template.php, theme-settings.php.

Also inside of the STARTERKIT folder, which is now your sub-theme's project folder, you'll see a css folder, where our style sheets can go, a js folder for our javascript, and an images folder for our images. You'll also see STARTERKIT.info.txt. This will become your theme's .info file. Remember, this is similar to the way that modules have a .info file. It tells Drupal some information about the files used by our theme, its name, etc.

  • STARTERKIT.info.txt is a template .info file for your sub-theme, so rename it to match the name of your theme - myzen.info. NOTE: Make sure to remove the .txt extension from the file, otherwise it will not show up on the Appearance page. On a mac, you may have to right click on the file, select "Get Info" and manually delete the .txt extension. Then open it up and edit the name and description fields. Read through the comments in the whole file, they explain very clearly what each line is used for.
  • At this point, you should be able to launch your site, go to the Appearance page and then click "Enable and set default" on your new sub-theme.
  • Inside of the .info file, add a stylesheet for your project to the end of the list of stylesheets. Use square bracket notation to append the name of your .css file to the end of the stylesheets array. Or, you can edit the included css/styles.css file. (We've made a copy of the STARTERKIT folder, which is what it is made for, so it doesn't count as hacking a contributed theme).
sites/all/themes/myzen/myzen.info
stylesheets[all][] = css/print.css
stylesheets[all][] = css/mystyles.css

Create your own CSS to override existing CSS

You can now add as many of your own CSS rules as you like to style the elements of your theme. Pause here and do some exploring with your Zen sub-theme. To get started digging in, watch this video and follow along with your Zen sub-theme. The video is a little long, but it does a really good job of demystifying the way Drupal is rendering your pages in the browser, and it will give you some good tips and tools for exploring themes and debugging your CSS.

Javascript!

We can add our own JavaScript files by putting them in the js folder and uncommenting out this line in our .info file. Then replace script.js with the name of your file.

sites/all/themes/myzen/myzen.info
scripts[] = js/script.js

As it is, this line adds the script.js file from inside of the js folder to the list of scripts, just as you could add your own .js file. We can also edit the file script.js. and put all the fancy jQuery in it we want.

Adding regions to a sub-theme

To add a region to a theme, you need to do two things. Assign it a name in the .info file:

sites/all/themes/myzen/myzen.info
...
regions[help]           = Help
regions[content]        = Content
regions[sidebar_first]  = First sidebar
regions[sidebar_second] = Second sidebar
regions[footer]         = Footer
regions[bottom]         = Page bottom
regions[mine] = My Region

The key in the square brackets is the machine readable name of the region, and the words to the right of the equals sign are the display name of the region. This will show up in the Blocks Administration page for your theme when you click on the link "Demonstrate block regions".

A couple things here - notice that the key does not need quotes. Also, these lines in the .info file don't need semicolons at the end. I know, I know, it's weird. I didn't make the rules. But this code still functions like the associative arrays we're used to.

Now that we've named the region, we need to tell Drupal where to display it. We do this by adding the region to the template file page.tpl.php. To get the file, go into the original zen folder, inside of templates, then copy the file page.tpl.php and paste it into your sub-theme folder under templates. We are overriding it, just as we were overriding the Core template files in the last lesson. Now we can edit the file and our changes will override the original file without having to directly modify it. Add this line to the bottom of the file:

sites/all/themes/myzen/templates/page.tpl.php
<?php print render($page['mine']); ?>

Use the machine readable name as the key for the $page array. We pass the $page array into Drupal's built in render function, similarly to how we would pass a twig file into the $app['twig']->render() method in Silex.

Now how do we use this region? Well, a region won't show up unless it has a block in it. We need to add and enable a block in the region for it to display. You can do this using the Block Administration page in the Drupal interface. You'll be able to see your region in the "Demonstrate block regions" view.
